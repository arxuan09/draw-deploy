## 后台配置速查：这些信息填到哪里

`gpt-image-2` 走 **`OpenAI /v1/images/generations`** 端点。该端点的模型表单模板是 `openai`（= 像素尺寸 + `auto`/`low`/`medium`/`high` 质量）。在「后台 → 模型管理 → 新增 / 编辑模型」中：

| 本文档里的概念 | 对应后台字段 | 填写说明 |
|---|---|---|
| 质量等级（Quality） | **图片质量选项**（image quality options） | 填入 `auto` `low` `medium` `high` |
| 质量 → 扣费 | **质量扣费映射**（quality credit mapping） | 每个质量等级单独定价；高质量成本更高 |
| 分辨率（Size） | **图片尺寸选项**（image size options） | 以 `宽x高` 字符串填入，如 `1024x1024` |
| 分辨率 → 扣费 | **图片尺寸扣费映射**（image size credit mapping） | 每个尺寸单独定价；高分辨率成本更高 |
| 模型标识 | **模型 slug** | 填上游模型名，如 `gpt-image-2` |

> 选择 **端点类型** 时选 `OpenAI /v1/images/generations`，表单的可视化选项编辑器会自动切换为「像素尺寸 + 质量」样式（跟随端点，而非供应商）。

## gpt-image-2

来源：OpenAI 官方文档（截至 2026 年 6 月最新指南）。

### 1. 支持的质量等级（Quality）

| 质量等级 | 英文参数值 | 说明 | 推荐使用场景 |
|----------|------------|------|--------------|
| 低质量   | `low`      | 生成速度最快、成本最低 | 快速草稿、缩略图、原型迭代 |
| 中等质量 | `medium`   | 速度与质量的平衡选项 | 大多数常规使用场景 |
| 高质量   | `high`     | 最高视觉质量 | 最终交付、专业设计、印刷用途 |
| 自动     | `auto`     | 默认值，模型自动选择最优质量 | 推荐大多数情况下使用 |

> **注意**：`quality` 参数仅对 GPT Image 模型（包括 `gpt-image-2`）有效。

### 2. 支持的分辨率（Size / Resolution）

`gpt-image-2` 支持**任意分辨率**（以 `"WIDTHxHEIGHT"` 字符串指定，例如 `"1536x864"`），但必须满足以下**硬性约束**：

**核心约束条件**

- 宽度和高度必须均为 **16 的整数倍**
- 长边与短边比例不得超过 **3:1**（即宽高比介于 1:3 与 3:1 之间）
- 单边最大长度 ≤ **3840 px**
- 总像素数范围：**655,360 ～ 8,294,400** 像素
- **2560×1440 以上**（约 368 万像素）的分辨率属于**实验性功能**，稳定性可能较低

**常用 / 推荐分辨率示例**

| 分辨率 (Size) | 方向 | 总像素数 | 备注 |
|---------------|------|----------|------|
| `1024x1024`   | 正方形 | 1,048,576 | 最常用、正方形基准 |
| `1536x1024`   | 横向 | 1,572,864 | 横向常用 |
| `1024x1536`   | 纵向 | 1,572,864 | 纵向常用 |
| `2048x2048`   | 正方形 | 4,194,304 | 2K 正方形 |
| `2048x1152`   | 横向 | 2,359,296 | 2K 横向 |
| `3840x2160`   | 横向 | 8,294,400 | 4K 横向（最大边长） |
| `2160x3840`   | 纵向 | 8,294,400 | 4K 纵向 |
| `auto`        | 自动 | — | 默认值（推荐） |

**API 请求示例**

```json
{
  "model": "gpt-image-2",
  "prompt": "...",
  "size": "1536x1024",
  "quality": "high"
}
```

**提示**

- 正方形图像通常生成速度最快。
- 超出约束范围的请求将返回错误。
- 实际生成时间与成本会随 `quality` 和 `size` 显著变化（高分辨率 + 高质量成本最高）。

## Gemini Nano Banana 系列

来源：Google Gemini API 官方文档（2026 年最新）。三个模型都走 **`Gemini Nano Banana generateContent`** 端点（端点模板 `gemini` = 宽高比 + 1K/2K/4K 分辨率分级）。

### 模型对比总览

| 模型名称 | Gemini API 标识符 | 支持的分辨率 (image_size) | 支持的比例数量 | 备注 |
|---|---|---|---|---|
| **Nano Banana** | `gemini-2.5-flash-image` | 仅 `"1K"` | 10 种 | 速度优先，仅支持基础 1K 分辨率 |
| **Nano Banana 2**（推荐） | `gemini-3.1-flash-image` | `"1K"`, `"2K"`, `"4K"` | **14 种** | 支持最全比例，推荐大多数场景 |
| **Nano Banana Pro** | `gemini-3-pro-image` | `"1K"`, `"2K"`, `"4K"` | **10 种** | 专业级，最高支持 4K |

> **默认值**：未指定时 `image_size` 默认 `"1K"`。

### 支持的宽高比（Aspect Ratio）

**Nano Banana 2（`gemini-3.1-flash-image`）—— 最全，14 种**

```text
"1:1", "1:4", "1:8", "2:3", "3:2", "3:4",
"4:1", "4:3", "4:5", "5:4", "8:1",
"9:16", "16:9", "21:9"
```

**Nano Banana Pro（`gemini-3-pro-image`）—— 10 种**

```text
"1:1", "2:3", "3:2", "3:4", "4:3", "4:5", "5:4",
"9:16", "16:9", "21:9"
```

**Nano Banana（`gemini-2.5-flash-image`）—— 10 种**

```text
"1:1", "2:3", "3:2", "3:4", "4:3", "4:5", "5:4",
"9:16", "16:9", "21:9"
```
### 支持的分辨率档位（Image Size）

| 模型 | 支持值 | 说明 |
|---|---|---|
| **Nano Banana** | `"1K"` | 固定 1K（约 1024px 级别），无其他选项 |
| **Nano Banana 2** | `"1K"`, `"2K"`, `"4K"` | 支持 1K/2K/4K 全分辨率 |
| **Nano Banana Pro** | `"1K"`, `"2K"`, `"4K"` | 高分辨率输出 |

> - 更高分辨率（2K/4K）会增加生成时间和 Token 消耗。
> - 实际像素尺寸按所选宽高比自动计算（例如 16:9 的 2K 约为 2752×1536）。

### 在本系统后台如何配置

gemini 模板下，**宽高比** 配在「图片尺寸选项」、**分辨率档位** 配在「图片质量选项」：

| 概念 | 对应后台字段 | 填写说明 |
|---|---|---|
| 端点类型 | **端点选择** | 选 `Gemini Nano Banana generateContent` |
| 模型 slug | **模型 slug** | 填 API 标识符，如 `gemini-3.1-flash-image` |
| 宽高比（Aspect Ratio） | **图片尺寸选项**（image size options） | 填**宽高比字符串**，如 `1:1` `16:9` `9:16` `21:9`；原样作为 `imageConfig.aspectRatio` 下发，任意 `N:M` 均可透传 |
| 分辨率档位 1K/2K/4K | **图片质量选项**（image quality options） | 直接用 `1K`/`2K`/`4K` 作质量值最直观（原样透传为 `imageConfig.imageSize`）；也兼容 `high`→4K、`medium`→2K、`low`/`auto`→1K |
| 比例 → 扣费 | **图片尺寸扣费映射** | 按宽高比定价 |
| 档位 → 扣费 | **质量扣费映射** | 按 1K/2K/4K 定价（高分辨率成本更高） |

> 字段映射在 `backend/internal/ai/registry.go` + `backend/internal/imagegen/adapters.go` 硬编码：`imageConfig.aspectRatio = getAspectRatio(尺寸选项)`、`imageConfig.imageSize = getGeminiImageSize(质量选项)`。后台无需关心 wire 字段名。

**注意事项（与上游能力的差异）**

- ⚠️ **分辨率档位随质量始终下发**：给只支持 `1K` 的 Nano Banana（`gemini-2.5-flash-image`）配 `2K`/`4K`，上游可能报错或被忽略 —— 按各模型实际支持范围设置质量选项。
- ⚠️ **极端比例按模型区分**：`1:8`/`8:1`/`1:4`/`4:1` 仅 Nano Banana 2 支持；给另外两个模型配这些比例可能报错。

## 通义万相 Wan 系列（DashScope）

来源：阿里云百炼「万相 2.7 图像生成与编辑」官方文档。走 **`DashScope 通义万相 image (异步)`** 端点（`AuthPreset: dashscope`，路径 `/api/v1/services/aigc/image-generation/generation`）。

> **⚠️ 异步端点**：与 OpenAI / Gemini「一次请求拿结果」不同，本端点上游是**异步任务流**——后端先创建任务（`X-DashScope-Async: enable`）拿 `task_id`，再轮询 `GET /api/v1/tasks/{task_id}` 至 `SUCCEEDED`，下载结果图（URL 有效期 24h）后转存到对象存储。这些都由后端适配器完成，**后台配置方式与同步端点完全一致**，无需特殊处理。

端点模板是 **`gemini`**（= 宽高比 + 1K/2K/4K 分级），所以后台模型表单与 Gemini 系列长得一样。

### 模型对比总览

| 模型标识（slug / model_key） | 简介 | 支持的分辨率档位 | 4K 支持 |
|---|---|---|---|
| **`wan2.7-image-pro`** | 专业版，质量更高 | `1K` / `2K`（默认）/ `4K` | ✅ **仅文生图（无输入图、非组图）** 支持 4K；图像编辑最高 2K |
| **`wan2.7-image`** | 速度更快 | `1K` / `2K`（默认） | ❌ 不支持 4K |

> 各档位总像素：**1K ≈ 1024×1024、2K ≈ 2048×2048、4K ≈ 4096×4096**。

### 分辨率与宽高比

`size` 参数支持两种**互斥**写法，本系统适配器会自动选择：

- **档位法（推荐）**：`1K` / `2K` / `4K`。文生图输出为**正方形**；有输入图时输出**自动匹配输入图的宽高比**并缩放到该档位。
- **像素法**：`宽*高`（**星号分隔**，如 `2048*1152`）。文生图总像素 `[768*768, 4096*4096]`、其他场景 `[768*768, 2048*2048]`，宽高比范围均为 `[1:8, 8:1]`。

本系统映射规则(`backend/internal/imagegen/dashscope.go`)：

- **质量选项**（1K/2K/4K）→ 决定 `size` 档位；
- **宽高比选项**(如 `16:9`)→ 仅文生图生效：适配器按「档位 + 比例」算出像素 `宽*高` 下发；方形 / `auto` 直接用档位；**极端比例在低档位下若低于 768×768 会自动回退方形档位**；
- **图像编辑**（带参考图）：忽略所配比例(上游按输入图比例缩放)，且 **4K 自动降为 2K**(编辑场景上游不支持 4K)；
- 实际输出尺寸以上游返回的 `usage.size` 为准，存库尺寸准确。

### 输入图片（图像编辑 / 多图参考）

- 端点 `SupportsReferenceImages = true`：支持传入参考图做**图像编辑 / 多图参考**(0–9 张，本系统按「编辑」处理，参考图经 SSRF 校验后以 base64 内联下发)。
- **不支持**红蒙版 inpaint(`SupportsInpaint = false`，万相用 bbox 框选式编辑，非蒙版)与 outpaint。
- 上游图片限制：JPEG/PNG(无透明通道)/BMP/WEBP，宽高 `[240, 8000]`、比例 `[1:8, 8:1]`、≤20MB。

### 在本系统后台如何配置

| 概念 | 对应后台字段 | 填写说明 |
|---|---|---|
| 供应商 | **供应商管理** | 新建供应商：Base URL = `https://dashscope.aliyuncs.com`(北京；新加坡用对应域名)，API Key = 百炼 `sk-xxx`。供应商 key 可自定义，鉴权/路径由端点决定 |
| 端点类型 | **端点选择** | 选 `DashScope 通义万相 image (异步)` |
| 模型 slug | **模型 slug / 模型 Key** | 填 `wan2.7-image-pro` 或 `wan2.7-image` |
| 分辨率档位 1K/2K/4K | **图片质量选项**（image quality options） | 用 `1K`/`2K`/`4K` 作质量值；也兼容 `high`→4K、`medium`→2K、`low`/`auto`→1K |
| 宽高比 | **图片尺寸选项**（image size options） | 填宽高比字符串如 `1:1` `16:9` `9:16`(仅影响文生图)；也可直接填像素 `1664x928`(系统自动转成万相的 `1664*928`) |
| 档位 → 扣费 | **质量扣费映射** | 按 1K/2K/4K 定价(高分辨率成本更高) |
| 比例 → 扣费 | **图片尺寸扣费映射** | 按宽高比定价 |
| `watermark` / `thinking_mode` / `seed` 等 | **高级默认参数**（defaultParams JSON） | 可选；原样并入 `parameters`(用户所选 `size`/`n` 优先) |

### ✍️ 选项栏到底填什么(复制即用)

**① 图片质量选项(= 分辨率档位)** —— 填这几个值:

```text
1K
2K
4K
```

> `4K` **只有 `wan2.7-image-pro` 的文生图**支持;配 `wan2.7-image` 时**只填 `1K` `2K`**,别填 `4K`。

**② 图片尺寸选项(= 宽高比)** —— 推荐填这一组常用比例(横/竖/方各覆盖):

```text
1:1
16:9
9:16
4:3
3:4
3:2
2:3
4:5
5:4
21:9
```

**填写规则:**

- 比例写成 `宽:高` 字符串,**任意 `N:M` 都能透传**,只要落在万相允许的 **`1:8` ～ `8:1`** 范围内(比如还想加 `2:1` `1:2` `3:1` `1:3` 都行)。
- 想要**精确像素**:把某一项直接填成 `宽x高`(如 `1664x928`),系统会自动转成万相的 `1664*928` 下发。注意**总像素**别超上限——**文生图(pro)≤ 4096×4096、编辑/组图及 `wan2.7-image` ≤ 2048×2048**,且比例仍需在 `1:8`～`8:1` 内。
- ⚠️ **宽高比/自定义像素只对「文生图」生效。** 走**图像编辑(带参考图)**时,输出比例由**输入图自动决定**(缩放到所选档位),这里配的比例会被忽略。
- 每个尺寸选项可在「**图片尺寸扣费映射**」里单独定价(比例本身不影响万相计费,万相按出图张数计费,但你可以借此对不同比例做差异化收费)。

> 选项填好后,用户在生成时就能从下拉里选「分辨率档位 + 宽高比」两项;系统把它们合成万相的 `parameters.size`(档位法或像素法)下发。

**注意事项**

- ⚠️ **本系统只取首图、固定 `n=1`**：不支持组图(`enable_sequential`)多图输出——即便上游返回多张也只采用第一张。
- ⚠️ **4K 仅 `wan2.7-image-pro` 的文生图**：给 `wan2.7-image` 配 4K、或在图像编辑中用 4K 都会被降级 / 报错，按模型与场景设置质量档位。
- ⚠️ **`thinking_mode` 默认开启**：仅在文生图(无输入图、非组图)时生效，会增加耗时但提升质量；可在默认参数里关。
- ⚠️ **异步耗时**：受上游排队影响，单次可能数十秒。后端轮询预算由 `generation_upstream_timeout_seconds`(默认 600s)控制，轮询期间会心跳刷新任务状态，避免被「超时回收」误判。
